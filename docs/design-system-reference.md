# Design System Reference

This document defines the official visual identity for the Supermarket project.

## Direction & Theme
- **Direction:** RTL (Arabic)
- **Theme:** Cashier-first design. Interfaces must be clean, practical, and highly functional over decorative elements.
- **Style Details:** Soft white cards, `rounded-2xl`, with clean and visible shadows.

## Standard Colors
- **Emerald** `#10b981`: For core actions, approvals, and completions.
- **Amber** `#f59e0b`: For alerts, suspensions (hold), and waiting states.
- **Rose** `#e11d48`: For cancellations, deletions, and error indicators.
- **Navy** `#1e3a8a`: For structure, headings, and persistent UI elements.
- **Background Base** near `#f8fafc`.

*Rule: Usage of colors must be strictly functional, not merely aesthetic.*

## Component Styles
- **Buttons**:
    - Primary buttons must be large and prominent for quick cashier actions.
    - Button semantics directly tie to the functional standard colors listed above.
- **Tables & Charts**:
    - Tables and charts are fundamental to the UI design; they must be clean, spacious, and highly legible.
- **Modals**:
    - Functional, focused overlays to manage contextual forms and actions quickly.
- **Responsive Behavior**:
    - Ensure adaptive scale, but optimize heavily for Cashier screens (which are typically wide landscape or touch-capable screens).
